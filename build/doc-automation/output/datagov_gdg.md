# Kurzbeschreibung
Das Gremium Datengouvernanz (DGD) ist das zentrale, bereichsübergreifende Entscheidungs- und Koordinationsgremium für Datenfragen. Die Mitglieder sind mandatiert und ihre Entscheidungen sind verbindlich.

Entscheidet und koordiniert bei Konflikten zwischen 🧑‍✈️🧭 Dateneigner (Data Owner).

# 1.2 Typische Entsatzfälle
- Zwei Bereiche nutzen unterschiedliche Definitionen (z. B. „Betrieb“). Soll zukünftig nur eine Definition gelten? Welche?
- Daten bzw. deren Qualität genügen nicht den Anforderungen anderer Organisationseinheiten
- Ressourcen müssen zwischen Bereichen abgestimmt werden
- Konflikte bei Datenzugriff oder Datenbereitstellung.

# 1.3 Entscheidungsprinzip
👉 Das Gremium Datengouvernanz (GDG) 👥🧭 entscheidet kollektiv. 
👉 Geschäftsführend: Data Management Lead 👤🧭

# 1.4 RACI
## Legende
- R (Responsible) = führt aus
- A (Accountable) = trägt Gesamtverantwortung
- C (Consulted) = wird einbezogen
- I (Informed) = wird informiert

| Aktivität | 🧑‍✈️🧭 Dateneigner (Direktbetroffene) | 👩‍💼💡 Datenverwalter | 🕵🏻⚙️ Datenhalter | Data Management Lead 👤🧭 | Gremium Datengouvernanz (GDG) 👥🧭
|---|---|---|---|---|---|
| Problem identifizieren | C | R | C | C | I |
| Analyse &amp; Optionen erarbeiten | C | R | C | R | I |
| Auswirkungen bewerten | C | R | C | R | I |
| Entscheidung vorbereiten |  I | C | C | R | C |
| Entscheid treffen |  C | I | I | C | A/R |
| Umsetzung koordinieren |  R | R | R | R | I |

# 1.5 Verantwortung des DGD
Sorgt für eine Abstimmung zwischen Digitalisierungsstrategie BLW und Umsetzung der BLW Datengouvernanz, koordiniert die Umsetzungsinitiativen und löst Konflikte bei der Standardisierung und Harmonisierung von Daten.

- Sicherstellung der bereichsübergreifenden Kohärenz
- Vermeidung von Daten-Silos
- Priorisierung im Sinne der Gesamtorganisation
